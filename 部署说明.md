# 校园食品交易网站 - 上线部署说明

## 一、你本地需要做的一次性准备

### 1. 打包

在项目根目录执行（需已安装 Java 17）：

```bash
./mvnw clean package -DskipTests
```

或使用脚本：

```bash
chmod +x deploy/build.sh
./deploy/build.sh
```

得到 **`target/campus-food-0.0.1-SNAPSHOT.jar`**，上传到服务器时可重命名为 **`campus-food.jar`**。

---

## 二、服务器上需要你手动完成的步骤

### 1. 安装环境

- **JDK 17**（Ubuntu 示例：`sudo apt update && sudo apt install openjdk-17-jdk -y`）
- **MySQL 8**（或 5.7+），并创建数据库和用户：

```bash
sudo mysql -u root -p
```

在 MySQL 里执行（请把 `你的强密码` 换成实际密码）：

```sql
CREATE DATABASE campus_food CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'campus_food'@'localhost' IDENTIFIED BY '你的强密码';
GRANT ALL PRIVILEGES ON campus_food.* TO 'campus_food'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

- （可选）**Nginx**：若要用 80 端口或 HTTPS，需安装 Nginx。

---

### 2. 创建目录并放置文件

在服务器上执行：

```bash
sudo mkdir -p /opt/campus-food/uploads/foods
sudo chown -R $USER:$USER /opt/campus-food
```

将以下文件放到服务器 **`/opt/campus-food/`** 下：

| 文件 | 说明 |
|------|------|
| `campus-food.jar` | 你本地打包好的 JAR（由 `target/campus-food-0.0.1-SNAPSHOT.jar` 重命名） |
| `application-prod.yml` | 生产配置（见下一步） |

---

### 3. 配置生产环境（必须手动修改）

复制示例配置并编辑：

```bash
cp /path/to/项目/deploy/application-prod.yml.example /opt/campus-food/application-prod.yml
nano /opt/campus-food/application-prod.yml
```

**必须修改的两处：**

1. **`spring.datasource.password`**：改成你在上面为 `campus_food` 用户设置的 MySQL 密码。
2. **`app.upload-dir`**：若上传目录不用 `/opt/campus-food/uploads/foods`，改成你实际路径（必须是绝对路径）。

保存后确认路径和密码无误。

---

### 4. 启动应用

**方式 A：直接运行（先测试用）**

```bash
cd /opt/campus-food
java -jar campus-food.jar --spring.profiles.active=prod --spring.config.additional-location=file:/opt/campus-food/application-prod.yml
```

浏览器访问 `http://服务器IP:8080`，能打开首页说明应用正常。用 `Ctrl+C` 停止。

**方式 B：用 systemd 长期运行（推荐）**

1. 编辑服务文件中的用户名：

```bash
sudo cp /path/to/项目/deploy/campus-food.service /etc/systemd/system/
sudo nano /etc/systemd/system/campus-food.service
```

把 **`YourUser`** 改成你用来运行服务的系统用户名（例如当前用户）。

2. 启用并启动：

```bash
sudo systemctl daemon-reload
sudo systemctl enable campus-food
sudo systemctl start campus-food
sudo systemctl status campus-food
```

查看日志：`journalctl -u campus-food -f`。

---

### 5. （可选）配置 Nginx 反向代理

若希望用 80 端口或域名访问：

1. 编辑 Nginx 配置，把 **`your-domain-or-ip`** 改成你的域名或服务器 IP：

```bash
sudo cp /path/to/项目/deploy/nginx.conf /etc/nginx/sites-available/campus-food
sudo nano /etc/nginx/sites-available/campus-food
```

2. 启用并重载：

```bash
sudo ln -s /etc/nginx/sites-available/campus-food /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

3. 若需 HTTPS，可安装 certbot：`sudo apt install certbot python3-certbot-nginx -y`，然后执行 `sudo certbot --nginx -d 你的域名`。

---

### 6. 防火墙 / 安全组

- 云服务器：在安全组中放行 **80**、**443**、**8080**（若直接访问 8080）。
- 本机防火墙示例：`sudo ufw allow 80 && sudo ufw allow 443 && sudo ufw allow 8080 && sudo ufw enable`。

## 四、部署相关文件说明

| 路径 | 说明 |
|------|------|
| `deploy/application-prod.yml.example` | 生产配置示例，复制到服务器并改名为 `application-prod.yml` 后修改密码和路径 |
| `deploy/campus-food.service` | systemd 服务文件，需把 `YourUser` 改为实际运行用户 |
| `deploy/nginx.conf` | Nginx 反向代理示例，需把 `your-domain-or-ip` 改为域名或 IP |
| `deploy/build.sh` | 本地打包脚本 |
| `src/main/resources/application-prod.yml` | 项目内嵌的生产配置模板（含占位密码），实际部署以服务器上的 `application-prod.yml` 为准 |

以上步骤中，**必须由你手动完成**的只有：安装 JDK/MySQL、创建数据库与用户、修改 `application-prod.yml` 中的密码和路径、在 systemd 里填写运行用户、在 Nginx 里填写域名或 IP，以及上线后注册第一个管理员。
